# ✅ heyflow 웹사이트 런칭 필수 체크리스트

헤이플로우가 제작하는 모든 웹사이트가 법적 문제를 예방하고 마케팅 효율을 극대화할 수 있도록, 런칭 전 반드시 확인해야 하는 체크리스트입니다. 새로운 프로젝트를 오픈할 때마다 이 문서를 확인해 주세요!

---

## 1. 🔒 개인정보 보호법 준수 (Contact/폼 영역)
웹사이트 내에 고객의 연락처, 이름 등을 수집하는 입력 폼(상담 신청, 회원가입 등)이 있다면 아래 사항을 반드시 확인해야 합니다.

- [ ] **동의 체크박스 유무:** 폼 하단에 `[필수] 개인정보 수집 및 이용에 동의합니다.`라는 체크박스가 존재하는가?
- [ ] **제출 차단 로직(Validation):** 사용자가 체크박스를 선택하지 않고 제출 버튼을 누르면, 알림창과 함께 제출이 차단되는가?
- [ ] **필수 4대 고지사항 명시:** 체크박스 주변에 아래 4가지 항목이 텍스트로 명확하게 노출되어 있는가?
  1. **수집 목적** (예: 프로젝트 문의 및 상담)
  2. **수집 항목** (예: 이름, 연락처 등)
  3. **보유 및 이용 기간** (예: 상담 종료 후 1년)
  4. **동의 거부 시 불이익** (예: 동의 거부 시 상담이 제한될 수 있습니다.)
- [ ] **전문 보기 링크 연동:** 상세한 '개인정보처리방침(Privacy Policy)'을 읽을 수 있는 별도 페이지가 존재하며, 폼에서 해당 페이지로 이동할 수 있는 링크가 있는가?

---

## 2. 🔎 검색엔진 최적화 (SEO) 기본 세팅
웹사이트가 완성된 후 구글과 네이버 검색에 빠르게 노출되기 위한 필수 조치입니다.

- [ ] **robots.txt / sitemap.xml 생성:** 검색 로봇이 사이트 구조를 파악할 수 있도록 루트 경로에 파일이 올바르게 생성되었는가? (전체 정식 도메인 주소 기반)
- [ ] **동적 데이터(포트폴리오 등) 자동화 사이트맵 연동:** 노션 등 외부 DB로 관리되는 개별 상세 페이지 주소들이 `sitemap.ts`를 통해 자동으로 구글에 꽂히도록 수집 연동이 완료되었는가?
- [ ] **상세 페이지 동적 메타데이터(generateMetadata) 적용:** 포트폴리오나 블로그 상세 페이지(`[id]/page.tsx`) 접속 시, 공통 제목이 아닌 **해당 게시물 고유의 제목과 설명글**이 브라우저 탭과 구글 검색 결과에 정확히 노출되도록 코드가 적용되었는가?
- [ ] **소유권 확인 메타 태그 삽입:** 네이버 서치어드바이저 및 구글 서치콘솔에서 발급받은 소유권 확인용 HTML 태그가 `<head>`에 잘 삽입되었는가?
- [ ] **포털 수집 요청:** 네이버/구글 콘솔에 접속하여 사이트맵(sitemap.xml) 제출 및 수집 요청 버튼을 클릭했는가?
- [ ] **오픈그래프(Open Graph) 세팅:** 카카오톡 등 외부 메신저로 링크를 공유했을 때, 웹사이트(또는 개별 포트폴리오)의 예쁜 대표 이미지(OG Image)와 설명글이 정상적으로 나타나는가?

## 3. 📱 크로스 브라우징 및 반응형 체크
- [ ] **모바일 환경 최적화:** 좁은 화면(768px 이하)에서 텍스트가 잘리거나, 스크롤 애니메이션(패럴랙스 등)이 의도치 않게 작동하여 가독성을 해치지 않는가?
- [ ] **안전 영역(Safe Area) 확보:** 스마트폰 노치나 하단 스와이프 바 영역에 중요한 버튼(예: 무료 상담하기)이 겹치지 않는가?
